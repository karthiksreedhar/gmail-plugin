# Quick Reply Templates

A Gmail Plugin feature that allows users to create and use pre-defined email response templates based on email categories. Templates support dynamic sender name replacement using the `[SENDER NAME]` placeholder.

## Overview

The Quick Reply Templates feature streamlines email responses by providing:

1. **Template Management**: Create, edit, and delete response templates for each email category
2. **Quick Reply Buttons**: One-click access to templates directly from email cards
3. **Dynamic Content**: Automatic sender name replacement using `[SENDER NAME]` placeholder
4. **Seamless Integration**: Works with existing response generation workflow

## Features

### Template Management
- **Category-based Templates**: Create unique templates for each email category
- **Template Editor**: Simple text area interface for each category
- **Real-time Status**: Visual indicators show which categories have templates
- **Bulk Operations**: Save all templates at once
- **Template Deletion**: Remove templates individually or by clearing content

### Quick Reply Integration
- **Smart Button Display**: Quick Reply buttons appear only on emails with categories that have templates
- **Hover Animation**: Buttons fade in/out on email hover for clean UI
- **Intelligent Placement**: Positioned between category pills and delete button
- **Direct Navigation**: Skips directly to response page with template applied

### Dynamic Content Processing
- **Sender Name Extraction**: Automatically extracts names from various email header formats
- **Flexible Parsing**: Handles "Name <email@domain.com>" and plain email formats
- **Fallback Handling**: Uses email username when display name unavailable
- **Template Processing**: Replaces all instances of `[SENDER NAME]` in templates

## Installation

The feature is automatically loaded when the server starts. No additional installation steps required.

## Usage

### Setting Up Templates

1. **Open Template Manager**
   - Click "Quick Reply Templates" button in the header menu bar
   - Modal opens showing all available email categories

2. **Create Templates**
   - Enter template text in the text area for each category
   - Use `[SENDER NAME]` placeholder where you want the sender's name inserted
   - Example: "Hello [SENDER NAME], thank you for your email regarding..."

3. **Save Templates**
   - Click "Save All Templates" to save all changes
   - Templates are saved per-user in the database
   - Status indicators show which categories have templates

4. **Delete Templates**
   - Click "Delete" button next to any category with a template
   - Or clear the text area and save to remove the template

### Using Quick Reply

1. **Locate Emails with Templates**
   - Quick Reply buttons appear on email cards that have categories with saved templates
   - Buttons are positioned between category pills and delete button
   - Buttons fade in on email hover

2. **Generate Quick Response**
   - Click "Quick Reply" button on any qualifying email
   - System automatically:
     - Extracts sender name from email headers
     - Applies appropriate template for the email's category
     - Replaces `[SENDER NAME]` with actual sender name
     - Opens response page with template as "generated" content

3. **Edit and Send**
   - Template appears in response area as if generated by AI
   - User can edit the response as needed
   - Use existing save/refine/copy functionality

## API Endpoints

All endpoints are prefixed with `/api/quick-reply-templates/` and require user authentication.

### GET `/api/quick-reply-templates/`

Retrieve all templates for the current user.

**Response:**
```json
{
  "success": true,
  "templates": {
    "Teaching & Student Support": "Hello [SENDER NAME], thank you for reaching out about your assignment...",
    "Research & Lab Work": "Hi [SENDER NAME], I appreciate your interest in our research..."
  },
  "updatedAt": "2025-01-15T10:30:00Z"
}
```

### POST `/api/quick-reply-templates/`

Save or update a template for a specific category.

**Request Body:**
```json
{
  "category": "Teaching & Student Support",
  "template": "Hello [SENDER NAME], thank you for your email regarding your assignment question..."
}
```

**Response:**
```json
{
  "success": true,
  "templates": { /* all templates */ },
  "category": "Teaching & Student Support",
  "message": "Template saved"
}
```

**Notes:**
- Empty template string deletes the template
- Category must be a valid existing category
- Template can contain `[SENDER NAME]` placeholders

### DELETE `/api/quick-reply-templates/:category`

Delete template for a specific category.

**Parameters:**
- `category` (URL parameter): Category name to delete template for

**Response:**
```json
{
  "success": true,
  "templates": { /* updated templates */ },
  "category": "Teaching & Student Support",
  "message": "Template deleted"
}
```

### POST `/api/quick-reply-templates/generate`

Generate response from template with sender name substitution.

**Request Body:**
```json
{
  "category": "Teaching & Student Support",
  "senderName": "John Doe",
  "senderEmail": "john.doe@example.com"
}
```

**Response:**
```json
{
  "success": true,
  "response": "Hello John Doe, thank you for your email regarding your assignment question...",
  "template": "Hello [SENDER NAME], thank you for your email regarding your assignment question...",
  "senderName": "John Doe",
  "category": "Teaching & Student Support",
  "justification": "Generated from Quick Reply template for \"Teaching & Student Support\" category"
}
```

## Data Storage

Templates are stored per-user in MongoDB collection `quick_reply_templates_data`:

## Troubleshooting

### Templates Not Saving
- **Check**: Ensure you have email categories set up first
- **Verify**: Check browser console for error messages
- **Retry**: Try saving individual templates instead of bulk save

### Quick Reply Buttons Not Appearing
- **Categories**: Ensure emails have categories assigned
- **Templates**: Verify templates exist for those categories
- **Reload**: Refresh the page to reload templates and re-render buttons

### Sender Name Not Replacing
- **Format**: Check that template contains `[SENDER NAME]` placeholder (case-sensitive)
- **Email Headers**: Verify email has proper from/originalFrom fields
- **Fallback**: System will use email username if display name unavailable

### Response Page Not Opening
- **Modal**: If inline composer fails, feature will fallback to modal view
- **Permissions**: Ensure feature has proper permissions in manifest
- **Console**: Check browser console for JavaScript errors

## Browser Compatibility

- **Modern Browsers**: Chrome, Firefox, Safari, Edge (latest versions)
- **JavaScript**: Requires ES6+ support for async/await
- **Clipboard**: Copy functionality requires secure context (HTTPS/localhost)

## Security Considerations

- **User Isolation**: Templates are stored per-user, no cross-user access
- **Input Sanitization**: Templates are sanitized before storage and display
- **XSS Protection**: Template content is properly escaped in UI
- **Authentication**: All API endpoints require user authentication

## Performance

- **Template Loading**: Templates loaded once on feature initialization
- **Button Rendering**: Efficient DOM manipulation, only processes visible emails
- **Caching**: Templates cached in memory to avoid repeated API calls
- **Debouncing**: Template status updates debounced to reduce server calls
